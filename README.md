electronic_diary
================

Електронния бележник съдържа следните функции:

1) Функции за манипулация на потребители:

- Стартирайте ElDiary.pl -nu USERNAME PASSWORD за създаване на нов потребител (-nu = new user)

- Стартирайте ElDiary.pl -du USERNAME PASSWORD за изтриване на потребител USERNAME с парола 

PASSWORD (-du = delete user)

- Стартирайте ElDiary.pl -s USERNAME PASSWORD за да стартирате електронния бележник със

съответния потребител и парола

2) Функционалности на Бележника

Целия бележник е направен с графичен интерфейс.

- Дясното меню представлява списък с файловете на дадения потребител (при създаване на нов 

потребител ще липсва). При избиране на файл от това менщ той се отваря в редактора.

- New, Save и Delete са ясни за какво са (Save и Delete изискват да бъде избран първо файл, ако 
такъв не е избран съобщават за грешка)

- За съжаление не ми остана време да направя Order By функционалността, но не мисля че тя е и 
необходима при наличието на графичен интерфейс (който не бях предвидил в началото, но ми стана

интересно да видя как работи).

- При затваряне на прозореца текущия файл се запазва автоматично. (Има малък бъг в това 
отношение: ако не е избран файл съобщава за грешка. Това е така защото използва същата функция 
като Save. Мога да го отстраня при нужда, но реших че не е голям проблем)

3) Файлове

- UI.pm това е модулът който съдържа функциите които създават UI-то, както и тези които се 
извикват от съответните бутони

- Users.pm - логване, създаване, изтриване на потребители

- Enc.pm - отговаря за криптирането на файловете

- Папките са папки на тестови потребители с техните файлове (всеки потребител има уникална 
папка)
